
### 5.1 第一阶段：完整手测（最高优先级）

用微信开发者工具跑通以下流程，记录 bug 到本文档后续条目：

   - 活动时间在未来 14 天内；
   - 发布 / 申请 / 同意时时间冲突（后端 409，前端是否有清晰提示）；
   - 草稿保存、编辑、删除；
   - 多账号切换后消息红点是否正确。

### 5.2 第二阶段：修阻塞 bug

优先修复影响演示的问题：

- 注册失败、发布失败、申请状态错乱。
- 聊天进不去、pending 与正式会话状态混乱。
- 联系方式提前暴露。
- 日程冲突漏判或前端无友好提示。
- 多账号切换红点异常（biggboss01 已修过一轮，需复测）。

### 5.3 第三阶段：UI 大改（按产品愿景）

建议顺序：

1. **广场卡片 + 详情页** — 第一印象，四分类强配色。
2. **onboarding** — 基础信息 / 自我介绍两段式、横向流转动效、欢迎页改为「欢迎来到 LuoJiaLink！」。
3. **消息页 + chat-room** — 微信式 inbox，pending / 正式会话视觉强区分。
4. **发布页** — 「创建校园计划」而非普通表单。
5. **我的页** — 个人中心层级。
6. **日程页** — 向苹果日历靠拢（月视图或格点 + 色块标记）。

### 5.4 第四阶段：体验补全

- 头像：微信 chooseAvatar / 相册选择，替代 URL 输入。
- 年级格式产品定稿：用户原始需求为 `2023/2022` 入学年，当前代码为 `大一/大二/研一`（biggboss01 刻意修改），需确认保留哪一种。
- 广场「全部」分类：biggboss01 已加且默认「全部」，用户原始描述为四个板块，需确认。
- 统一冲突弹窗 UX（发布、申请、同意三处）。
- 文案全面重写：去工程化，改成学生社交语气。
- Logo、空状态、加载态、轻量页面过渡动画（布局重做之后再做）。

### 5.5 第五阶段：可选增强（验收后）

## 2026-06-14 修复 MBTI 选项选中高亮

## 2026-06-14 AI 聊天破冰功能

### 做了什么

- **破冰助手卡片**：正式聊搭会话（`session`）顶部展示「AI 破冰助手」，含推荐时间、推荐地点、当前开场白。
- **智能开场白**：根据双方昵称、帖子标题/分类、地点、时间、共同兴趣等上下文生成多条中文开场白；默认使用本地模板（`AI_MODE=template`），配置 OpenAI 后可切换为真实 AI 生成。
- **填入聊天框**：一键把当前开场白写入输入框，用户可编辑后发送。
- **换一句**：调用刷新接口轮换下一条开场白，避免只会背一句。
- **来源标识**：卡片上显示「智能模板」或「AI 破冰」，便于区分当前生成方式。

### 改了哪些文件

- `server/src/services/icebreaker.ts`（新建）
- `server/src/config/env.ts`
- `server/.env.example`
- `server/src/routes/recommendations.ts`
- `server/tests/recommendation-report.test.ts`
- `miniprogram/pages/chat-room/chat-room.js`
- `miniprogram/pages/chat-room/chat-room.wxml`
- `miniprogram/pages/chat-room/chat-room.wxss`
- `docs/20260614shikaibo1.md`

### 接口说明

- `GET /api/recommendations/session/:id`：响应新增 `icebreakers[]`、`sourceLabel`。
- `POST /api/recommendations/session/:id/icebreaker/refresh`：请求体 `{ current }`，返回下一条 `icebreaker`。

### 可选 AI 配置

在 `server/.env` 中设置：

```
AI_MODE=openai
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=https://api.openai.com/v1   # 可选，默认即此
OPENAI_MODEL=gpt-4o-mini                    # 可选
```

未配置或 AI 调用失败时自动回退到模板开场白。
