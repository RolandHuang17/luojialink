# Step 03：会话消息与阶段留痕

更新时间：2026-06-13

## 1. 本轮开发前已经实现了什么

截至 Step 03 开始前，项目已经完成本地 MVP 的前半条主链路和匹配关系闭环：

- 可以使用 `alice`、`bob`、`carol` 进行 mock 登录。
- 可以查看和维护用户资料、标签。
- 可以在广场查看未过期结伴请求。
- 可以发布结伴请求并查看请求详情。
- 可以点击“想搭”提交申请。
- 发起者可以查看收到的申请，并确认或拒绝。
- 确认申请后，系统会创建临时会话。
- 双方已经能在会话页看到匹配成功后的会话列表。
- 已有 `docs/STEP_02_APPLICATION_SESSION.md` 记录申请与会话创建阶段。

当时的核心缺口是：会话只能被看到，不能真正发送和查看消息。

## 2. 本轮开发实现了什么

本轮目标是让临时会话具备最基础的文字沟通能力。

### 2.1 后端新增能力

新增会话消息接口：

- `GET /api/sessions/:id/messages`：获取会话消息列表。
- `POST /api/sessions/:id/messages`：发送文本消息。
- `POST /api/sessions/:id/close`：关闭会话。

新增或完善的业务规则：

- 只有会话双方可以读取消息。
- 只有会话双方可以发送消息。
- 非会话成员访问消息接口会被拒绝。
- 会话关闭后不能继续发送消息。
- 消息当前只支持文本，`contentType` 固定为 `text`。
- 消息按创建时间正序返回，便于小程序直接渲染。

### 2.2 小程序新增能力

会话页从静态列表升级为可交互页面：

- 展示我的会话列表。
- 点击某个会话后加载该会话消息。
- 展示消息发送方和消息气泡。
- 区分自己发送和对方发送的消息。
- 支持输入文本并发送。
- 发送成功后清空输入框，并刷新消息列表和会话最近消息。
- 无消息时展示“暂无消息”空状态。

### 2.3 测试覆盖

后端集成测试继续覆盖完整链路：

- Alice 和 Bob 匹配后都能看到会话。
- Alice 可以在会话中发送文本消息。
- Bob 可以读取 Alice 发送的消息。
- Carol 不是会话成员，不能读取该会话消息。
- 会话关闭后，Alice 不能继续发送消息。

## 3. 当前可演示流程

现在可以演示：

1. 启动后端并打开微信开发者工具。
2. 登录 `alice`，发布一条结伴请求。
3. 切换到 `bob`，在广场找到该请求并点击“想搭”。
4. 切回 `alice`，在“我的 - 匹配申请”中确认 Bob。
5. Alice 进入“会话”页，选择新会话。
6. Alice 发送一条文本消息。
7. 切换到 Bob，进入“会话”页，选择同一会话。
8. Bob 可以看到 Alice 的消息，并继续发送文本消息。

这意味着 P0 主流程已经具备“发布、申请、确认、会话、发消息”的核心闭环。

## 4. 后续规划

下一轮建议优先做：

- 日历时间片：保存用户空闲时间。
- 共同空闲：根据双方时间片和请求时间计算交集。
- 冲突提示：提示用户同一时间段已有安排。
- AI/模板建议：基于活动类型、地点、标签生成破冰建议。
- POI 地点：查询本地 mock 地点并用于推荐。
- 举报页完善：从小程序提交真实举报记录。
- 演示脚本：编写 `docs/DEMO_SCRIPT.md`，固定验收演示路径。
- 测试清单：编写 `docs/TEST_CASES.md`，方便验收前逐项自测。

## 5. 本轮不做的内容

- 不做 WebSocket 实时推送。
- 不做图片、语音或文件消息。
- 不做消息撤回、已读回执。
- 不做真实 AI 接口调用。
- 不做复杂会话详情页拆分。
- 不处理微信开发者工具自动生成的本地配置文件。
