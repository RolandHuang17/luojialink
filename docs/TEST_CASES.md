# 验收前测试清单

更新时间：2026-06-14（Step 06 完整闭环版）

## 1. 后端自动化测试

在 `server/` 目录执行：

```bash
npm run build
npm test
```

通过标准：

- TypeScript 编译无错误。
- Vitest 全部通过（5 个测试文件、8 个用例）。
- 不需要真实 AI Key。
- 不需要真实地图服务。

## 2. 启动检查

```bash
cd server
npx prisma db push
npm run db:seed
npm run dev
```

浏览器访问：

```text
http://127.0.0.1:3000/api/health
```

通过标准：

- 返回 `code: 0`。
- `data.ok` 为 `true`。

小程序：

- 微信开发者工具导入 `miniprogram/`。
- 开启「不校验合法域名」。

## 3. 登录与注册 onboarding

- [ ] `alice` / `bob` / `carol` 可登录并进入广场。
- [ ] `newbie` 登录后进入 onboarding，未完成前不能进入其他页面。
- [ ] onboarding 全部必填项可提交，完成后显示欢迎页并进入广场。
- [ ] 注册字段锁死选项（学院、年级、年龄、性别、家乡、常住、MBTI、关系、特质）不可随意输入。
- [ ] MBTI 四题使用引导问题，界面不暴露 I/E/S/N 等字母。

## 4. 广场与详情

- [ ] 广场可加载帖子列表，四分类（及「全部」）可切换。
- [ ] 卡片展示发布者、标题、时间、地点。
- [ ] 详情页展示完整规划，可进入发布者主页。
- [ ] 非发布者可点击「想搭」；发布者本人按钮禁用。

## 5. 发布与草稿

- [ ] 发布页必填校验：标题、具体规划、活动地点。
- [ ] 可发布到吃饭/运动/自习/娱乐分类。
- [ ] 可保存草稿，在「我的 → 草稿箱」继续编辑或删除。
- [ ] 同时有效想搭不超过 5 个。
- [ ] 活动时间须在未来 14 天内。
- [ ] 与已有发布/申请/已搭上活动冲突时，发布被拦截并弹出友好提示。

## 6. 申请与匹配

- [ ] 不能申请自己发布的想搭。
- [ ] 可申请他人想搭，重复申请会失败。
- [ ] 申请出现在消息页，pending 状态不能自由聊天。
- [ ] 发布者可同意/拒绝；申请者可撤回。
- [ ] 与已有安排时间冲突时，申请被拦截并弹出友好提示。
- [ ] 同意后创建正式会话，双方可在消息页进入聊天窗口。

## 7. 会话消息与联系方式

- [ ] 正式会话可发送和接收文本消息。
- [ ] 非会话成员看不到他人消息。
- [ ] 会话关闭后不能继续发送消息。
- [ ] 正式会话页展示模板破冰建议。
- [ ] 双方点击「互换联系方式」后，才展示对方微信号。
- [ ] 个人主页中他人微信号默认隐藏。

## 8. 日程

- [ ] 日程页展示近 14 天活动。
- [ ] 红色=我发布，蓝色=我申请，绿色=已搭上。
- [ ] 点击活动可进入详情。
- [ ] 同意申请时若双方时间冲突，操作被拦截并提示。

## 9. 我的与资料

- [ ] 「我的」展示有效想搭数量 `activeCount / 5`。
- [ ] 可管理已发布想搭、取消发布。
- [ ] 点击资料卡可预览个人主页。
- [ ] 编辑资料页可修改资料；MBTI 通过「重新测试」修改。

## 10. 消息红点（本地逻辑）

- [ ] 新申请/新会话在消息 Tab 显示红点。
- [ ] 切换 A/B/C 账号后红点不互相污染。
- [ ] A 同意 B 的申请后，B 侧消息 Tab 出现红点。

## 11. 举报

- [ ] 帖子详情、他人个人主页可进入举报页。
- [ ] 举报可提交成功；非法目标 ID 不可提交。

## 12. Git 检查

提交前执行：

```bash
git status --short
```

不应提交：

- `.env`
- `server/prisma/*.db`
- `node_modules/`
- `dist/`
- `miniprogram/project.private.config.json`
