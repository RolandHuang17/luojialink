# Step 02：想搭申请、申请处理与会话创建

更新时间：2026-06-13

## 1. 本轮开发前已经实现了什么

本轮开始前，项目已经具备本地 MVP 的基础骨架：

- 微信小程序原生前端已经能导入微信开发者工具并运行。
- 后端 `Node.js + TypeScript + Express + Prisma + SQLite` 已经能本地启动。
- 已有 mock 登录，支持 `alice`、`bob`、`carol` 三个演示账号。
- 已有用户资料、标签列表、标签保存接口。
- 已有广场列表、发布请求、请求详情、取消请求接口。
- 已有基础举报创建和演示用举报查看接口。
- 已有 Prisma schema、SQLite 初始化脚本和 seed 数据。
- 小程序已接入登录、广场、发布、请求详情、我的资料页面。

当时的核心缺口是：用户只能发布和浏览请求，还不能真正发起“想搭”、处理申请并形成会话。

## 2. 本轮开发实现了什么

本轮目标是打通 P0 主链路的后半段：申请、确认、创建会话。

### 2.1 后端新增能力

新增申请相关接口：

- `POST /api/posts/:id/applications`：提交想搭申请。
- `GET /api/posts/:id/applications`：发起者查看某条请求收到的申请。
- `GET /api/applications/me`：查看我发出的申请。
- `GET /api/applications/received`：查看我收到的申请。
- `POST /api/applications/:id/accept`：确认申请。
- `POST /api/applications/:id/reject`：拒绝申请。

新增会话相关接口：

- `GET /api/sessions`：查看我的临时会话列表。

新增或完善的业务规则：

- 不能申请自己的请求。
- 不能重复申请同一条请求。
- 不能申请已过期或非 `published` 状态的请求。
- 只有请求发布者可以查看和处理该请求的申请。
- 确认申请后，申请状态变为 `accepted`，请求状态变为 `matched`，并创建 `TempSession`。
- 同一请求的其他待处理申请会自动变为 `rejected`。
- 申请列表只返回有限用户资料，不返回实名和学号。
- 匹配分采用基础算法：`min(95, 60 + 10 * commonTagCount)`。

### 2.2 小程序新增能力

请求详情页：

- “想搭”按钮已接入真实申请接口。
- 申请成功后给出明确反馈。

申请页：

- 展示“我收到的申请”。
- 展示申请者有限资料、共同标签、匹配分和申请状态。
- 支持确认和拒绝申请。
- 展示“我发出的申请”和处理状态。

会话页：

- 展示匹配成功后的临时会话列表。
- 展示对方昵称、学院、请求内容、地点和费用。
- 暂时保留静态 AI 建议卡片，真实 AI 建议接口放到后续开发。

我的页：

- 增加“匹配申请”入口，方便演示时进入申请处理页。

### 2.3 测试覆盖

新增后端集成测试覆盖主链路：

- Bob 不能申请自己的请求。
- Bob 可以申请 Alice 的请求。
- Bob 不能重复申请同一请求。
- 非发布者不能查看或处理申请。
- Alice 可以查看收到的申请。
- Alice 确认申请后创建会话。
- Alice 和 Bob 都能在会话列表看到该会话。

## 3. 当前可演示流程

现在可以演示：

1. 启动后端。
2. 打开微信开发者工具。
3. 登录 `alice`。
4. Alice 发布结伴请求。
5. 切换登录 `bob`。
6. Bob 在广场查看请求详情。
7. Bob 点击“想搭”。
8. 切回 `alice`。
9. Alice 在“我的 - 匹配申请”中看到 Bob 的申请。
10. Alice 点击确认。
11. Alice 和 Bob 都能在“会话”页看到新会话。

这意味着项目已经从“发布信息板”推进到“可形成匹配关系”的本地 MVP。

## 4. 后续规划

下一轮建议优先做：

- 会话消息：消息列表、发送文本、仅双方可访问。
- 日历时间片：保存空闲时间、展示共同空闲、提示冲突。
- AI/模板建议接口：根据活动类型、地点、标签生成破冰建议。
- POI 地点接口：查询 seed 中的本地 mock 地点。
- 举报页完善：从小程序提交真实举报。
- 演示脚本：补 `docs/DEMO_SCRIPT.md`，固定 10 分钟演示路径。
- 测试用例：补 `docs/TEST_CASES.md`，记录验收前自测清单。

## 5. 本轮不做的内容

- 不做真实消息发送。
- 不做 WebSocket。
- 不做真实 AI API。
- 不做真实地图或点评 API。
- 不做正式上线配置。
- 不处理微信开发者工具自动生成的本地配置文件。
