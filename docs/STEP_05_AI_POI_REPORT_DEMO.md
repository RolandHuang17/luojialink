# Step 05：模板 AI 建议、POI 推荐、举报闭环与演示留痕

更新时间：2026-06-13

## 1. 本轮开发前已实现

截至 Step 05 开始前，项目已经完成本地 MVP 主链路：

- mock 登录：`alice`、`bob`、`carol`。
- 请求广场、发布请求、请求详情。
- “想搭”申请、发起者确认/拒绝申请。
- 确认后创建临时会话。
- 会话列表、消息列表、文本消息发送、关闭会话。
- 日历时间片维护、共同空闲计算、忙碌冲突提示后端接口。
- 阶段文档：`STEP_02_APPLICATION_SESSION.md`、`STEP_03_CHAT_MESSAGES.md`、`STEP_04_CALENDAR.md`。

当时主要缺口是：会话里的 AI 建议仍是静态文案，POI 数据没有查询接口，举报页还是占位，验收演示脚本和测试清单尚未固化。

## 2. 本轮开发实现

### 2.1 后端能力

新增接口：

- `GET /api/pois?category=&keyword=`：查询本地 mock POI，按类别、关键词、评分和相关性排序。
- `GET /api/recommendations/session/:id`：仅会话双方可访问，基于会话请求、共同空闲、POI 和双方标签生成模板建议。

推荐结果包含：

- `timeSuggestion`：优先使用双方共同空闲，没有共同空闲时回退到请求时间。
- `placeSuggestion`：优先使用本地 POI，没有匹配 POI 时回退到发布者地点偏好。
- `icebreaker`：根据活动类型和共同标签生成破冰话术。
- `notes`：说明建议来源、共同空闲状态、会话状态。

继续保留并测试：

- `POST /api/reports`：创建举报记录。

本轮不改数据库 schema，不接真实 AI Key，不接真实地图服务。

### 2.2 小程序能力

会话页：

- 选择会话后加载真实模板 AI 建议。
- 展示建议时间、建议地点、破冰话术、注意事项。
- 展示推荐 POI 列表。
- 增加“举报会话”入口。

请求详情页：

- 进入详情后调用冲突检测接口。
- 如果当前用户忙碌时间片与请求时间重叠，展示提示。
- 冲突只提示，不阻止用户继续“想搭”。
- 增加“举报请求”入口。

举报页：

- 从占位页升级为可提交表单。
- 支持目标类型、目标 ID、举报原因、补充说明。
- 提交到 `/api/reports`，成功后返回上一页。

## 3. 验证结果

后端验证：

- `npm run build` 通过。
- `npm test` 通过。

新增测试覆盖：

- POI 可按类别和关键词查询。
- 会话成员可获取模板推荐。
- 非会话成员不能获取推荐。
- 推荐结果包含时间建议、地点建议、破冰话术。
- 举报接口可创建记录，非法参数返回 400。

小程序手测建议：

1. Alice 发布请求。
2. Bob 查看详情，如有忙碌重叠则看到冲突提示。
3. Bob 点击“想搭”。
4. Alice 确认申请，创建会话。
5. Alice/Bob 进入会话页，选择会话。
6. 查看共同空闲、智能建议、推荐地点。
7. 从详情页或会话页提交举报。

## 4. 当前可演示闭环

现在可以完整演示：

1. 登录。
2. 发布结伴请求。
3. 查看详情和时间冲突提示。
4. 提交“想搭”申请。
5. 处理申请并创建会话。
6. 在会话中发送消息。
7. 查看共同空闲。
8. 查看模板 AI 建议和 POI 推荐。
9. 提交举报反馈。

这已经覆盖 5 天验收演示需要的核心产品故事：从匹配、沟通、时间协调、智能建议到风险反馈。

## 5. 后续规划

下一阶段建议优先做：

- UI 打磨：统一页面文案、空状态、按钮层级和视觉密度。
- 演示数据稳定化：固定 seed 场景，减少现场手动准备。
- 管理端轻量补强：查看举报列表和处理状态。
- README 更新：补启动、测试、演示入口。
- 可选增强：真实 AI 接口、地图 SDK、WebSocket、订阅提醒。

验收前不建议大改主链路，也不建议引入真实外部依赖。
