# Step 06：完整可演示产品闭环升级

更新时间：2026-06-14

## 1. 开发前状态

项目此前已经具备本地 MVP 主链路：

- mock 登录、广场、发布、详情。
- “想搭”申请、确认/拒绝、创建临时会话。
- 会话消息、关闭会话。
- 日历时间片、共同空闲、冲突提示。
- 模板 AI 建议、POI 推荐、举报提交。

主要问题：

- 用户资料仍是早期简化字段。
- 缺少首次注册 onboarding。
- 发布和帖子字段不符合“找搭子/找朋友”产品表达。
- pending 申请和正式聊天没有清晰区分。
- 微信号隐私和互换联系方式没有闭环。
- 前端多处中文文案和结构需要统一。

## 2. 本轮实现

### 2.1 数据模型和规则

- 扩展 `User`，补齐头像、年龄、性别、家乡、微信号、常驻校区、MBTI、关系期待、自我介绍、爱好、喜欢的三件事、给对方的话、雷点、个人特质、onboarding 状态。
- 扩展 `Post`，新增标题、具体规划、活动地点，保留旧字段做兼容映射。
- 扩展申请/会话，支持申请撤回和联系方式互换状态。
- 新增后端锁死选项常量，SQLite 下由 Zod 做枚举校验。
- 新增统一时间规则服务，覆盖 5 个有效发布、未来 14 天、发布/申请/接受申请冲突检查。

### 2.2 后端接口

- 用户：选项下发、onboarding 提交、资料更新、个人主页。
- 帖子：广场分类筛选、发布、保存草稿、更新草稿、删除草稿、我的发布、草稿箱。
- 申请：创建、撤回、通过、拒绝。
- 消息：统一消息列表，pending 申请和 matched 会话同页展示。
- 联系方式：双方成功搭上并互换后才显示微信号。
- 日程：新增 `/api/calendar/events`，返回发布、申请、已搭上三类活动。

### 2.3 小程序页面

- 新增 onboarding 流式注册页面。
- 原生 tabBar 改为五入口：广场、日程、+、消息、我的。
- 广场按吃饭、运动、自习、娱乐筛选，卡片只展示摘要。
- 发布页支持发布和保存草稿。
- 详情页展示完整规划、发布者主页入口、想搭、冲突提示、举报。
- 消息页区分 pending 申请窗口和正式聊天窗口。
- 日程页展示近两周活动，用红/蓝/绿区分发布、申请、已搭上。
- 我的页支持发布管理、草稿箱、资料编辑、主页预览。
- 新增个人主页和草稿箱页面。

## 3. 验证结果

后端：

- `npm run build` 通过。
- `npm test` 通过，5 个测试文件、8 个测试用例全部通过。

小程序基础检查：

- 所有 `miniprogram/pages/**/*.js` 通过 `node --check`。
- `app.json` 可正常解析。

新增测试覆盖：

- onboarding 枚举和个人特质数量校验。
- 有效发布最多 5 个。
- 活动时间必须在未来 14 天内。
- 发布冲突校验。
- 申请冲突校验。
- pending 申请进入消息列表。
- accepted 后可聊天。
- 微信号默认隐藏，双方互换后可见。

## 4. 产品取舍

- 未引入自定义 tabBar，先使用原生五入口，并将中间发布入口显示为 `+`，优先保证稳定。
- 未接真实微信登录、真实地图、真实 AI，继续使用本地 mock 和模板能力。
- 旧字段 `description/locationPref/feePref` 暂时保留，用于兼容已有页面和推荐接口。
- 日程页先做近两周视图，不做复杂拖拽日历。
