# 验收演示脚本

更新时间：2026-06-13

## 目标

用 10 分钟展示 LuoJia Link 的本地 MVP：校园结伴请求从发布、申请、匹配、会话、日历协调、智能建议到举报反馈的闭环。

## 演示前准备

后端：

```bash
cd server
npm run db:seed
npm run dev
```

小程序：

- 用微信开发者工具导入 `miniprogram/`。
- 开启“不校验合法域名”。
- 确认后端地址为 `http://127.0.0.1:3000/api`。

建议账号：

- Alice：发布者。
- Bob：申请者。
- Carol：非会话成员，用于说明权限隔离。

## 10 分钟流程

### 1. 登录和广场

1. 选择 Alice 登录。
2. 进入广场，说明当前是校内结伴请求列表。
3. 展示请求包含活动类型、地点、费用和匿名发布者。

### 2. 发布请求

1. 进入发布页。
2. 发布一条晚饭、自习或运动请求。
3. 回到广场确认新请求出现。

### 3. 日历准备

1. Alice 进入日历页，保存一个 `available` 时间片。
2. 切换 Bob 登录，保存一个与 Alice 有重叠的 `available` 时间片。
3. 可额外给 Bob 保存一个 `busy` 时间片，用于详情页冲突提示。

### 4. 想搭申请

1. Bob 打开 Alice 的请求详情页。
2. 展示时间冲突提示：系统提醒但不阻止操作。
3. Bob 点击“想搭”。

### 5. 申请处理

1. 切回 Alice。
2. 进入申请页，查看收到的申请。
3. 展示申请者有限资料、共同标签和匹配分。
4. Alice 点击确认。

### 6. 会话和消息

1. Alice 进入会话页，选择新会话。
2. 发送一条消息。
3. 切回 Bob，进入会话页，看到 Alice 的消息。
4. Bob 回复消息。

### 7. 共同空闲和智能建议

1. 在会话页展示共同空闲时间。
2. 展示智能建议：
   - 推荐时间。
   - 推荐地点。
   - 破冰话术。
   - 注意事项。
3. 展示推荐 POI 列表，说明当前使用本地 mock POI，可离线稳定演示。

### 8. 举报反馈

1. 从会话页点击“举报会话”。
2. 选择举报原因，填写补充说明。
3. 提交并看到成功提示。

## 讲解重点

- 本项目当前是本地演示版，不依赖上线、不依赖真实微信登录、不依赖真实 AI Key。
- 主链路已经闭环：发布、申请、确认、会话、消息、日历、推荐、举报。
- 匿名展示和有限资料降低陌生社交风险。
- 共同空闲和模板 AI 建议增强匹配后的推进效率。

## 现场故障处理

- 后端无法连接：确认 `npm run dev` 正在运行，并访问 `http://127.0.0.1:3000/api/health`。
- 数据不干净：执行 `npm run db:seed` 重置演示数据。
- 小程序请求失败：确认微信开发者工具已开启“不校验合法域名”。
- 页面没有新会话：确认 Alice 已接受 Bob 的申请。
