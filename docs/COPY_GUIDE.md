# LuoJiaLink 文案规范

更新时间：2026-06-14  
适用：珞珈连点微信小程序 MVP

## 1. 品牌与语气

- **品牌名**：界面主标题用 **LuoJiaLink**；副标题或解释可用「珞珈连点」。
- **称呼用户**：你 / 同学，不用「用户」「账号 holder」。
- **语气**：轻松、真诚、像武大同学说话；有温度，但不油、不卖萌过度。
- **场景感**：强调「找搭子、一起完成一个小计划」，不是填表、不是后台系统。

## 2. 禁止事项

- 不用工程词：`参数错误`、`mock`、`Token`、`接口`、`ID`（对用户可见处）。
- 不暴露 MBTI 字母（E/I/S/N/T/F/J/P）给用户看。
- 不把英文状态词直接展示（`pending`、`published`、`matched` 等需映射中文）。
- 不写长段业务说明；页头引导控制在 **一行**。

## 3. 常用词汇表

| 推荐 | 避免 |
| --- | --- |
| 想搭 / 一起 | 申请参与、匹配请求 |
| 搭上 / 确认同行 | matched、accepted |
| 招募中 | published |
| 等待对方 | pending（申请者视角） |
| 待你确认 | pending（发布者视角） |
| 互换联系方式 | exchange contact |
| 草稿 | draft |
| 演示登录 | mock 登录 |

## 4. 错误提示原则

- 先说人话，再说怎么办。
- 时间冲突：弹窗，标题「时间冲突」，正文说明撞了哪类安排。
- 网络/服务：「连不上服务器，请确认后端已启动」。
- 表单：「还有几项没填好」比「参数错误」好。

## 5. 空状态原则

- 一句话说明「现在没有啥」+ 一句轻引导「你可以……」。
- 例：「这里还没有想搭，不如发一条试试？」
